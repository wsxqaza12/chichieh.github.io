AWS Summit × Agile Taipei：兩場爆滿的 Agent Memory 分享

2026/07/21 · 活動 · 閱讀約 1 分鐘

上週四連續跑了 AWS Summit Taipei 跟 Agile Taipei 兩場分享，沒想到兩場都大爆滿，老實說人比我預期的多滿多 😅 真的很謝謝特地排時間過來聽的朋友，在台下看到不少熟面孔也是蠻開心的。

這次主要講的，是我最近一直在研究跟踩坑的 Agent Memory。

其實現在大家越來越意識到 Memory 對 Agent 有多重要，但說白了「Memory」這個詞包山包海，從當前對話狀態、專案知識、過去的任務經驗，到怎麼跨 Session 甚至在團隊間共享，全部都混在一起談。

同樣都打著 Memory 的招牌，你去對比 Claude Code、Codex、OpenClaw 或是 Mem0 就會發現，大家真正處理的重點其實不太一樣。

我個人覺得最有趣的是，個人用的 Agent Memory 解決的主要是連續性 (continuity)，但當這套記憶要落地到團隊層級，你真正要面對的就變成來源、範圍、審核、衝突，甚至得開始處理 Agent 的「遺忘」機制。

這次也是我第一次把最近啃的這些研究，還有 Cairn 真正想解的「人跟 Agent 到底要怎麼共享團隊記憶」整理成一場完整的分享。現場被問了不少有趣的問題，但也讓我更確定這題目蠻值得繼續深挖下去 💪

最近因為這題目讀了一堆 paper 還有產品文件，之後會再整理成幾篇文慢慢跟大家分享，不曉得大家最近在實作 Agent 時有沒有碰到什麼 Memory 相關的痛點？歡迎留言跟我交流一下 🙌 最後也再次謝謝 AWS 跟 Agile Taipei 的邀請～
