AIAgent正在慢慢對齊的一週

2026/01/18 · 趨勢談 · 閱讀約 2 分鐘

延續上週寫完那篇之後，這一週的新聞沒有讓我又震撼一次的感覺，反而是多了一種踏實感，很多事情都沒有變，但整體趨勢變得更清楚了。我感受到 Agent 這技術線，正在從各家各說各話，慢慢收斂成一套大家願意一起用的做法。 最明顯的，是 Agent Skills 這件事終於往開放標準走，Anthropic 把技能規格公開，總算不再綁在單一產品裡，後面 OpenAI 的 Codex 馬上跟上，其他工具也都開始支援。這其實是一個很重要但不太好講成 headline 的轉折，跟去年 MCP 一樣，這代表大家開始承認 Agent 的價值不在於模型有多強，而在於技能能不能被重用，被組合，被搬來搬去，這才是工程與產品面重要的事。 如果你有真的在做 agent，就會知道技能這層一直很痛，每個平台一套定義，每次重寫一次，最後大家都在重造輪子，上半年我就造了不少輪子。現在開始有人願意說我們先把格式對齊，剩下再各自發揮，這是一種趨向成熟的市場才會出現的選擇。 同一時間，Google 在做的事情其實也很一致，Chrome 讓 agent 直接進 DevTools 看問題自己修，直接示範什麼叫做 agent 真的嵌進日常工具裡。不再需要像以前一樣跳出一個聊天框告訴你哪裡怪怪的，而是看完錯誤後直接動手改，這種習慣一旦養成，使用者對 agent 的期待就回不去了。 Claude 的 Task Mode 我也覺得很值得觀察，它不是一個突然很厲害的新能力，而是把計畫、技能、工具使用全都放在同一個工作空間裡，讓人可以中途插話、改方向。這其實是在嘗試解一個老問題，當 agent 做到一半，我到底是在旁邊看，還是能一起工作，同樣這種互動模式如果順了，會影響很多人怎麼設計 agent UI。 我個人覺得比較容易被忽略的，是設計與開發工具開始全面 agent 化。Google Stitch 的平行編輯，TypeScript 的 Agent Development Kit，這些都在把 agent 拉回工程師熟悉的世界。型別、安全、模組化、平行處理等等，這些都不是新概念，但當它們出現在 agent 框架裡，就代表 agent 不再是實驗專案，而是要被長期維護的系統。 Google 放出的 2026 agent 趨勢報告，我反而是沒有覺得太在意，整體預測並不新鮮，但它選擇把 agent 放進核心流程、資安與訓練體系，這件事本身就是立場表態，對他們來說，agent 已經不是附加功能，而是下一個操作層，最近他們的新年聖誕禮包也是在做布局。 這周沒有什麼爆點，但我反而比較想把這些時刻記下來，因為過往真正改變世界的技術，往往不是在最吵的那一週，而是在大家開始默默對齊的時候。
