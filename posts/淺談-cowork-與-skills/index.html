淺談 Cowork 與 Skills

2026/01/16 · 技術 · 閱讀約 3 分鐘

這一週我看 AI Agent 的更新，感覺兩條主線變得更清楚了，第一條是由 Claude Cowork 打響的，把 Agent 做成桌面上的工作夥伴；第二條則把 Skills 做成可攜帶可安裝的模組，開始有一點套件生態系的感覺。

先講 Cowork桌面工作這條線，Anthropic 推出 Claude Cowork，我覺得它關鍵不在於又更會聊天，而是它用一個很直覺的方式把 Agent 放進你的檔案系統。你只要指定一個資料夾，Claude 就可以在那個範圍內讀寫檔案，自己規劃步驟，做完再把成果整理好，例如把截圖整理成試算表，或把散落筆記整理成初稿，這讓非工程師第一次有機會用到類似 Claude Code 的工作模式，感受到 AI 長出手來幫你做事。

但 Claude Cowork 只給 Max 用戶，太貴又不支援其他模型，因此一周內 Karan Vaidya 就拚出了一個開源版的 ComposioHQ/open-claude-cowork；後續 Github 又陸續有很多開源 Openwork 專案，逼得下放到 Pro 用戶 。 我研究了一下開源 Openwork 專案並列了幾個，後續有時間再測試給大家更具體的建議：

  1. ComposioHQ/open-claude-cowork
  2. accomplish-ai/openwork
  3. langchain-ai/openwork
  4. different-ai/openwork
  5. eigent-ai/eigent

Claude Cowork 或說 Openwork 的出現，代表一般大眾對於 AI Agent 的理解與應用，不再侷限在對話框裡給建議，而是能在電腦上把事情做完，而且能用人類可以接受的方式回報進度。我滿期待當這條線跑通，Agent 就會開始像同事，而不只是工具。

除卻 Openwork 之後，這周第二條主線是 Skills

這週 Skills 我反而覺得更集中，Anthropic 在 2025/10 就先提出 Agent Skills 的概念，在 2025/12 又把它整理成 open standard — Agent Skills format。

Google 的 Antigravity 在 2026/01/14 這波更新中宣布導入 Skills 並遵循 Agent Skills format；Vercel 則把自家多年 React / Next.js 的最佳化經驗整理成遵循 Agent Skills 格式的 agent-skills，並提供 add-skill / skills 這類安裝工具與目錄，讓 skills 變成可分發、可版本化的工程資產。有趣的是，以 npm weekly downloads 來看，Vercel 巨人一出來馬上就超越早期開源的 numman-ali/openskills 下載量

把 Antigravity 和 Vercel 放一起看，我覺得 skills 這個詞開始變得很具體。一個 skill 本質上就是一個資料夾，裡面至少有一份 SKILL.md（通常還會搭配 scripts / references 等資源），用來告訴 agent「遇到什麼情境要怎麼做、要遵守哪些規則、要用哪些工具與步驟」。

以前大家說 skills，常常像 prompt 模板或小技巧，現在更像是一個可以被安裝的模組，有規格，有目錄結構，有 metadata，變成一個可以版本化、可以交接、可以被多個 agent 重複使用的工作資產。

這會直接影響未來 agent 怎麼擴充，怎麼交接，怎麼讓團隊把經驗留下來，而不是每次重寫輪子。

Anthropic 在醫療和生命科學也推出新的 Agent Skills，包括 FHIR 開發和事前授權審查的範例 skill。當 skills 開始往醫療這種流程密度高又需要對齊規範的領域擴張，其實是在提醒大家，Agent 要進企業，最後一定繞不開治理和護欄。

這週如果要收斂成一句話，我會總結成 Agent 正在同時變得更像桌面同事，也變得更像可維護的工程系統。Cowork 和 OpenWork 解決的是「Agent 怎麼在桌面把事情做完」，Skills 解決的是「Agent 做事時要帶哪一套工作方法」。

預期它們會是接下來半年大家真正會用到的基礎設施。
