Vibe Coding 工具

2025/08/15 · 技術 · 閱讀約 1 分鐘

最近密集地使用了各種 vibe coding 工具，網頁設計的 v0、lovable、Bolt，code 的 cursor、codex、augment 等等。

深刻的感覺到，這些工具能幫你很快達到初階的目標。但當你繼續深度開發時，如果沒有良好的開發經驗，使用 vibe coding 會讓你進入一種「看似神速，實則混亂」的狀態，之後 debug 反倒要花非常大量的時間🫠。

以我開發平台的經驗來說，在使用 vibe coding 前，建議一定要把 User Flow、System Architecture、Data Schema、API Spec 等等先規劃好，這樣你才能下達準確的 Prompt，千萬別以為 vibe coding 能通靈幫你處理好 😂 (我有幾個功能頁面就是偷懶，debug 花的時間比從頭手刻還痛苦 )

因此以目前的 vibe coding 效果，我認為短時間工程師還是不會很快被取代，尤其是那些能夠做系統設計、架構規劃的人，通靈能力還是大家的優勢之一呀 😆

題外話 v0 現在的收費模式實在很迷，大量用戶都遷移了，目前我使用 curosr(或augment) + shadcn 能達到差不多的效果，不曉得大家都搬到哪裡去了?
