實測推薦 agent-broker：讓 Agent 們在 Discord 溝通

2026/04/06 · 技術 · 閱讀約 1 分鐘

最近把 Pahud Hsieh 開源的 agent-broker 實際跑起來，老實說真的蠻好用的，總算解決了讓「龍蝦團」跟「產品端」的 AI 能在 Discord 上流暢互相溝通的問題 🎉

之前為了串接嘗試過 Claude Code 的 Channel 功能，不過踩坑後發現有點像玩具，主要是它一次只能開一個 session，彈性跟實用性都不太夠 😅

換成 agent-broker 後體驗就好很多，輕量又非常容易部署。大家可以看圖片，我的龍蝦 CTO Agent 跟我討論完之後，就可以直接從 Discord 到產品端的 Channel 去叫負責寫 code 的 AI bot 起來幹活，還會自動開一個 thread 繼續敲 code，整個討論跟開發的過程都透明有據。

我個人覺得最棒的一點是，這種架構也順便實現了環境的隔離，像是 .env 這種敏感的金鑰與環境變數就可以確實留在產品端，不用跟龍蝦團的 Agent 混在一起，安全又乾淨 💪

其實當我們用到多個 Agent 分工時，真的會需要這種輕便的 broker 來幫不同職能的 bot 牽線，有興趣推薦去試試看這個專案 🔥

不曉得大家現在都是怎麼管理多個 Agent 之間的對話跟協作呢？歡迎留言跟我分享你的做法 👇
