Agent 經濟體正在成形

2026/03/26 · 趨勢談 · 閱讀約 3 分鐘

這兩週還有一條比較不技術但我覺得很重要的線——Agent 不只是在技術上變成熟，連商業與經濟層面的基礎建設也開始長出來了。

先講一個觀點，a16z 的 Sam Broner 寫了一篇關於 agent 支付的文章，核心論點是：agent 不會刷信用卡，它們會像企業一樣運作，有供應商條款、有信用額度。他認為可程式化的 stablecoin 可能會變成 agent-native commerce 的核心支付軌道。這聽起來很遠，但如果你想一下，當 agent 可以自己跑任務、自己呼叫 API、自己決定用哪個服務，它確實需要一套不依賴人類手動授權的支付方式。

Uniswap 的動作也在這條線上，它開放了 Developer Platform beta，讓開發者可以生成 API key 來加 swap 跟 LP 功能。更有意思的是它推了一個 Uniswap Skill，讓 agent 可以直接把 DeFi 的操作整合進 workflow 裡。agent + DeFi 這個組合之前大家都在喊，但現在真的有人把介面跟工具做出來了。

回到比較主流的企業端，Google Cloud 在 BigQuery 跟 Vertex AI 推了 Data Agents，讓團隊可以部署現成的 data agent 或用 ADK 自己建。企業分析正在從 dashboard 變成 agent 執行。OpenAI 也擴充了 Responses API 的檔案支援，agent 現在可以吃 docx、pptx、csv、xlsx 等格式。這些聽起來很 boring，但這就是 enterprise adoption 真正需要的東西——agent 能不能讀你公司的檔案、跑你的報表、接你的資料源。

Claude Sonnet 4.6 的更新裡有一個容易被忽略的細節，它整合了 S&P、LSEG、Moody’s、FactSet 這些金融數據源的 Excel 功能。這不是在秀模型能力，這是在告訴金融業：你們的 agent 可以直接接你們的資料了。

安全跟信任這塊也在跟上。Vercel 對 Skills 做了自動化安全審計，找了 Snyk、GenDigital、Socket 做獨立報告，涵蓋超過 6 萬個 skill。惡意 skill 會被隱藏，風險等級會直接顯示。這其實跟 app store 的審核機制很像，當 agent skill 變成一個生態系，就需要有人來做品質跟安全把關。

Gemini 開始在 Android 上做背景任務也是一個訊號。它不是讓你跟 AI 聊天，而是讓 agent 直接從 OS 層執行預訂、購買這類多步驟任務。Google 是直接把 agent workflow 嵌進作業系統裡，一旦使用者習慣這種操作模式，agent 就變成手機的一部分了。

Anthropic 收購 Vercept 也值得提一下，Vercept 做的是感知跟互動，被收進來是為了推進 Claude 的 computer use 能力。Sonnet 4.6 在 OSWorld 已經拿到 72.5%，離人類水準越來越近。當 agent 真的能操作軟體介面、完成交易、填表單，這些能力跟支付系統一接上，agent economy 就不再只是概念了。

我個人覺得 2026 下半年開始，大家討論 agent 的方式會從「它能做什麼」轉向「它花多少錢、誰付錢、怎麼結算」。技術成熟到一定程度之後，商業模式跟經濟基礎設施才是真正決定 agent 能不能規模化的關鍵。

歡迎大家分享你的看法，你覺得 agent 的商業模式會長什麼樣子？
