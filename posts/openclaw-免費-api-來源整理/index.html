跑 OpenClaw 不一定要花錢——我目前在用的幾個 API 來源

最近身邊越來越多人開始玩 OpenClaw，但聊到 API 來源大家就卡住了。自己本地跑模型硬體門檻高，直接拿 Claude Code 或 Gemini CLI 的帳號去反代又有封號風險，Anthropic 跟 Google 今年都動手封過一波了。

其實我覺得這個問題沒有想像中難解決，目前有好幾條路可以走，有些完全免費，有些一個月花不到一杯咖啡的錢。我自己實際在用的組合跑下來也還算穩定，這邊整理出來給大家參考。

先講我在選 API 來源時在意的三件事：模型能力不能太弱（7B 等級跑 OpenClaw 沒什麼意義），資料安全要有基本保障（OpenClaw 會接觸到個人資訊，來路不明的 API 風險太大），還有不能違反服務條款導致被封號。

完全免費的選項

Groq——速度快到有點誇張

先講這個是因為我自己日常用最多的免費來源之一。Groq 本來是一間做 AI 晶片的 startup，創辦人是前 Google TPU 首席架構師，2025 年底被 NVIDIA 收購了。被收購之後 API 反而更穩了。

他們晶片最大的特色就是快，GPT-OSS 跑出來可以到 500 token/s，體感上回應幾乎是瞬間出來的。免費 tier 不用信用卡，支援 Kimi k2、GPT-OSS、Llama 3.3 70B 這些開源模型，每天 1,000 次呼叫、30 RPM，個人用量很夠。

我自己的感覺是，寫程式相關的任務 Kimi k2 表現比較好，通用問答的話 GPT-OSS 比較順。

到 console.groq.com 註冊之後，進 API Keys 頁面建一把 gsk_ 開頭的 key 就能開始用了。

NVIDIA NIM——大模型也能白嫖

NVIDIA 的 Developer Program 免費開放了不少模型的 API，包括 Kimi k2.5（1 trillion 參數，目前開源模型裡的第一梯隊）跟 GPT-OSS-120b（OpenAI 開源的 117B MoE 模型）。

官方說 QPM 上限是 40，但實際體驗跟模型大小有關。k2.5 這種超大模型偶爾會抽風，100B 左右的反而跑得比較順。

註冊只需要信箱加手機號，到 build.nvidia.com 申請 NVIDIA Developer 帳號，找到想用的模型，點 Deploy → Get API Key，驗證完手機號就能拿到 nvapi- 開頭的 key。

花一點小錢，體驗大幅提升

OpenRouter——$10 解鎖高配額

OpenRouter 是一個 API 聚合平台，很多模型提供者把 API 放上去，使用者統一從 OpenRouter 呼叫。好處是選擇多，包括一些比較冷門的模型都找得到。

免費帳戶 20 RPM、每天 50 次。但只要往帳戶裡充 $10 就能解鎖到每天 1,000 次。這 $10 是存在帳戶裡的餘額，免費模型不會扣，純粹當解鎖門檻用，不是月費。

到 openrouter.ai 註冊，Settings > API Keys 建 key（sk-or- 開頭），想要高配額的話到 Credits 頁面充值就好。

Google AI Pro——context window 打遍天下無敵手

這條路稍微複雜一點，但我覺得性價比很高，特別是需要長 context 的任務。

Google AI Pro 會員每月 $19.99，附帶 Google Developer Program Premium 資格，每月送 $10 的 Google Cloud credits。這筆 credit 可以直接拿來呼叫 Gemini API（AI Studio 或 Vertex AI 都行），因為走的是付費 tier，速率限制遠比免費帳號寬鬆——Gemini 2.5 Pro 大約 150 RPM / 2,000 RPD，Flash 更高。

$10 聽起來不多，但以 Gemini 2.5 Flash 的定價來算（input $0.15/M tokens，output $0.60/M tokens），可以跑超過 1,500 萬 input tokens。日常用 OpenClaw 做輕量任務綽綽有餘，關鍵是 Gemini 的 1M context window 是其他家目前拼不過的。

設定步驟：訂閱 AI Pro → 到 Cloud Console 設結算帳號 → 到 Google Developer Program 頁面啟用 Premium（綁定結算帳號）→ 在 AI Studio 建 API Key 並開啟付費 tier。Credit 每月刷新，一年內有效。

順便提一個我聽說但還沒自己試過的玩法：AI 帳號可以加家庭群組，每個帳號最多五個家庭成員，每個成員的 credit 都能累積到同一個結算帳戶。理論上 credit 可以翻五倍。

封號的事大家都在擔心，但 Google 目前封的主要是濫用免費 tier 跟跑反代的帳號。透過正規 AI Pro 訂閱走付費 API 不違反 TOS，到目前為止沒看到被封的案例。

灰色地帶的選項：OpenAI Codex OAuth

最後這條要特別說明風險。用 ChatGPT 訂閱（Plus $20/月起）可以透過 OAuth 呼叫 Codex 模型，消耗的是訂閱配額而不是按 token 計費。這條線路在安裝 OpenClaw 的時候透過設定精靈完成，不用另外設定 API key。

OpenClaw 走的是跟 Codex CLI 同一套 OAuth 認證，社群很多人在用。但老實說，OpenAI 並沒有白紙黑字寫「第三方工具可以用 ChatGPT 訂閱跑自動化」。Anthropic 在 2026 年初已經封掉了 Claude 訂閱的第三方 OAuth，OpenAI 目前沒動作，但誰也說不準以後會不會跟進。

我的建議是拿來當輔助就好，不要 24/7 跑批量任務，也不要把它當唯一依賴。

我自己怎麼搭配的

我目前的組合是 Groq + NVIDIA NIM 處理日常任務，兩個都免費，速度跟穩定度都還行。需要吃長 context 的時候切到 Google AI Pro 的 Gemini，畢竟 1M 的 context window 目前沒有替代品。

說白了現在跑 OpenClaw 的 API 成本可以壓得很低，真正麻煩的反而是怎麼把多個來源串起來做 fallback，某一家掛了能自動切換，這塊之後有機會再聊。

大家如果有在用其他不錯的 API 來源歡迎留言分享，之後有空我也想整理一篇 OpenClaw 從零開始的設定教學。
