見到偶像 Pahud：AI Agent 趨勢與 MCP 的商業模式

2026/04/05 · 活動 · 閱讀約 2 分鐘

週四我特別推掉了一個客戶的會議，跑去內湖見了我的偶像 Pahud。老實說，這趟去得超值。題外話，本人真的比網路上的那個好像黏土(?)的頭像帥太多了。

聽 Pahud 分享他一生的經驗，其實真的蠻收穫良多的。

其中最有感的是整個 AI Agent 未來的趨勢，以及把抽象層能力打包成 MCP 階層的商業模式。就像這半年來大家常講的：The future of software is not just human-friendly, but agent-operable. 比起現在常用的 Skill 設計，MCP 確實在商業角度上更優雅地解決了原始碼外露的問題。

這是不是最終的最優解？我個人覺得還不好說，這邊我也還在邊試邊研究。但我認為不論如何，能為 AI Agent 提供更好 Infra 基礎建設的方向，絕對會越來越香，肯定也是未來各家爭鳴的主戰場。

另外，Pahud 提到他現在怎麼讓 Coding CLI 跟龍蝦在 Discord 上協作，這點真的讓我大開眼界 🤯

我也一直在思考怎麼讓我產品端的 Coding Agent 與龍蝦在 Discord 上協作，試過了各種方法（像是 Claude Code Channel 與各種開源），結果 CCC 根本就是玩具。我也想過自己要不要來寫，結果就剛好聽到了這個分享，實在太感謝了。本來趁著聽完這股熱血，我照著 Pahud 提到的概念正打算自己手刻一個，結果沒想到隔天人家直接開源了：agent-broker，想當然我馬上就去點了 Star，推薦大家多多支持。

這個專案真正解決的痛點，就是把 Coding CLI on the Cloud 這個概念給落地。

它其實是把一個超輕量、用 Rust 寫的 Broker（Pahud 提到這東西只吃 14MB memory，有夠省的XD），跟你的 Coding CLI 封裝在「同一個」Cloud Pod 沙箱裡面。這意味著什麼？這代表這個沙盒可以直接丟到遙遠的 Zeabur 去跑，它自己就能獨立對接 Discord！

你再也不用把 Coding Agent 逼得跟你原有的 OpenClaw Gateway 綁在同一台機器上了。總算可以實現「產品端」與「龍蝦這種自動化權限破表的 Agent」的全面物理隔離。不用擔心它會弄崩你的主機配置，你只要在 Discord 裡下個指令，它就自動幫你起一個全新的 thread，整個操作體驗順滑無比 🤣

最後，Pahud 大大在現場還順便介紹了四腿票這東西，那個飛行規劃超級誘惑…搞得我都想入坑了，簡直是現場推坑大會 XD

簡而言之，這是一個收穫良多的粉絲見面會!? 期待下次還有機會去到類似的場合 🤣
